# Keycloak Client Onboarder Skill

Stateless atomic operation to provision, configure, and secure single sign-on (SSO) client definitions inside the primary Keycloak Identity Provider realm.

## Prerequisites

- `keycloak-mcp` (or Keycloak Admin REST API tools) — for direct management of realms, clients, client scopes, and keys.

## Steps

### Step 1: Resolve Keycloak Realm
Target the appropriate administration realm context:
- Default target realm: `homelab`.
- Ensure connection context is initialized and verified.

### Step 2: Provision Client Definition
Create a new OIDC client definition with production-grade security defaults:
- **Client ID**: `<service-name>`
- **Protocol**: `openid-connect`
- **Access Type**: `confidential` (or `public` depending on application design)
- **Standard Flow**: Enabled (Authorization Code Flow)
- **Redirect URIs**: Include exact ingress routes: `https://<service-name>.arpa/login/oauth2/code/keycloak` and `https://<service-name>.arpa/*`
- **Web Origins**: `https://<service-name>.arpa`

### Step 3: Extract & Store Client Credentials
Extract secrets and credentials securely:
- Retrieve client ID secrets generated by Keycloak.
- Save credentials or pass them down securely to stack environment variables (using OpenBao or directly).
- Verify single sign-on redirection targets resolve successfully.
